## Priebeh stretnutia

### 1. Export pipeline dát

Do systému bola doplnená funkcionalita exportu pipeline dát vrátane potrebných API endpointov a frontendových komponentov. Tým sa umožňuje nadviazať na anotácie a ich ďalšie využitie mimo samotného rozhrania aplikácie.

### 2. Mock symbol matching a review výstupov

Pribudla aj mock funkcionalita pre symbol matching, ktorá rozširovala review časť a pripravovala projekt na budúce inteligentnejšie priraďovanie symbolov alebo kľúčov.

### 3. Perzistencia stavu pipeline

Do pipeline bola doplnená perzistencia stavu, vďaka čomu si systém pamätá priebeh rozpracovanej práce. Ide o dôležitú funkcionalitu pre reálne použitie pri dlhšej práci s dokumentmi.

### 4. UI úpravy používateľskej časti

Na frontende prebehli úpravy login a register časti, ako aj vylepšenia sekcií „Your Documents“ a „User Profile“. Tieto zmeny zlepšujú čitateľnosť rozhrania a zjednocujú používateľskú skúsenosť naprieč aplikáciou.

## Prijaté rozhodnutia

- Pokračovať v dopracovaní exportných možností ako súčasti pipeline.
- Zachovať perzistenciu stavu ako kľúčovú funkcionalitu pri viacstupňovom spracovaní.
- Priebežnými UI úpravami zjednocovať používateľskú časť aplikácie.
