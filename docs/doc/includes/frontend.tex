\section{Implementácia frontendu}

Frontendová časť systému bola implementovaná ako samostatná webová aplikácia postavená na knižnici React. Frontend zabezpečuje používateľské rozhranie pre celý pracovný postup: registráciu, prihlásenie, správu dokumentov a najmä šesťkrokové pipeline rozhranie pre poloautomatické spracovanie historických šifrovacích dokumentov. Komunikácia s backendovou službou prebieha výhradne cez REST API.

Z hľadiska tímového projektu frontendová časť tvorí samostatný modul vyvíjaný nezávisle od backendovej a AI časti systému. Takéto rozdelenie umožňuje paralelný vývoj jednotlivých komponentov, jednoduchšie testovanie a bezproblémové nasadenie aplikácie v kontajnerizovanom prostredí.

\subsection{Použité technológie}

Frontend bol implementovaný pomocou knižnice \textbf{React}~\cite{react} s využitím \textbf{React Router} pre navigáciu, vďaka čomu je aplikácia implementovaná ako Single Page Application (SPA) bez potreby opätovného načítania stránky. Na zobrazovanie používateľských notifikácií je použitá knižnica \textbf{React-Toastify}. Ikony v rozhraní sú z knižníc \textbf{React Icons} (sady Material Design Icons a Feather Icons). Build a vývojové skripty aplikácie sú riešené prostredníctvom \textbf{Create React App} (balík react-scripts). Všetky závislosti sú definované v súbore \texttt{package.json}.

\subsection{Architektúra frontendu}

Frontend je navrhnutý ako SPA aplikácia. Hlavný komponent \texttt{App} spravuje autentifikačný stav používateľa, definuje trasy a obmedzuje prístup k chráneným stránkam pre neprihlásených používateľov. Pre pipeline route (\texttt{/upload}) je render zabalený do špeciálneho plnoobrazovkového layoutu bez bočnej karty; pre ostatné trasy je obsah zobrazený v centrovanej karte.

Globálny stav pipeline (aktuálny krok, obrázok, bounding boxy, výsledky) je riadený cez React Context (\texttt{PipelineContext} a hook \texttt{usePipelineStep}), čo umožňuje zdieľanie stavu naprieč všetkými krokmi bez \textit{prop drilling}.

Autentifikačný token (JWT) je po prihlásení uložený do \texttt{localStorage} a pri každom API volaní je odosielaný v hlavičke \texttt{Authorization: Bearer <token>}. Pri načítaní aplikácie sa token overuje volaním \texttt{GET /me}; ak je token neplatný, používateľ je odhlásený.

\subsection{Smerovanie a riadenie prístupu}

Trasy aplikácie sú definované pomocou React Router:

\begin{itemize}
    \item \texttt{/} – domovská stránka,
    \item \texttt{/register} – registrácia (len neprihlásený),
    \item \texttt{/login} – prihlásenie (len neprihlásený),
    \item \texttt{/upload} – pipeline rozhranie (len prihlásený),
    \item \texttt{/documents} – správa dokumentov (len prihlásený),
    \item \texttt{/profile} – profil používateľa (len prihlásený),
    \item \texttt{/admin/users} – správa používateľov (len admin),
    \item \texttt{/admin/documents} – správa dokumentov (len admin).
\end{itemize}

\subsection{Implementácia používateľských funkcionalít}

\subsubsection{Domovská stránka}

Komponent \texttt{Home} slúži ako úvodný informačný pohľad. Zobrazuje privítanie a spracúva jednorazové notifikácie po registrácii alebo prihlásení pomocou \texttt{sessionStorage} a React-Toastify.

% [PLACEHOLDER: screenshot domovskej stránky — frontend_home.png]

\subsubsection{Registrácia a prihlásenie}

Komponent \texttt{Register} implementuje klientsku validáciu vstupov (min. 3 alfanumerické znaky pre meno, min. 8 znakov s číslicou pre heslo). Po úspešnej registrácii je odoslaný overovací e-mail; aktivácia účtu je podmienkou prihlásenia.

Komponent \texttt{Login} realizuje prihlasovanie emailom/heslom aj Google OAuth. Pri Google prihlásení backend vráti JWT token ako URL parameter, ktorý frontend zachytí pri načítaní a uloží do \texttt{localStorage}.

% [PLACEHOLDER: screenshot registrácie — frontend_register.png]
% [PLACEHOLDER: screenshot prihlásenia — frontend_login.png]

\subsubsection{Profil používateľa}

Komponent \texttt{Profile} umožňuje zmenu používateľského mena a hesla (vyžaduje potvrdenie starého hesla) a zobrazuje históriu aktivít (prihlásenia, nahrania, úpravy dokumentov) načítanú z endpointu \texttt{GET /me/activity}.

\subsubsection{Správa dokumentov}

Komponent \texttt{DocumentsList} zobrazuje tri sekcie dokumentov: vlastné (\textit{My Documents}), zdieľané (\textit{Shared with me}) a verejné (\textit{Public documents}). Pre každý dokument je možné:
\begin{itemize}
    \item otvoriť ho v pipeline rozhraní (tlačidlo \textit{Open in Pipeline}),
    \item stiahnuť pôvodný súbor,
    \item zmeniť úroveň prístupu (\texttt{private} / \texttt{public} / \texttt{shared}),
    \item zdieľať s konkrétnym používateľom vyhľadaným podľa mena,
    \item odstrániť (s potvrdením v modálnom okne).
\end{itemize}
Vyhľadávanie prebieha lokálne filtrovaním načítaných dokumentov podľa názvu.

% [PLACEHOLDER: screenshot správy dokumentov — frontend_documents.png]

\subsubsection{Administrácia}

Pre používateľov s rolou \texttt{admin} sú k dispozícii stránky \texttt{AdminUsers} a \texttt{AdminDocuments}, ktoré poskytujú prehľad a správu všetkých používateľov a dokumentov v systéme.

\subsection{Pipeline rozhranie}

Jadrom frontendu je \textbf{šesťkrokové pipeline rozhranie} pre spracovanie šifrovacích dokumentov, dostupné na trase \texttt{/upload}. Jednotlivé kroky sú zobrazené v hornej lište (\texttt{PipelineStepsBar}); po dokončení kroku sa odomkne nasledujúci a používateľ sa môže vrátiť k ľubovoľnému odomknutému kroku.

Všetky kroky zdieľajú dvojstĺpcový layout: vľavo je plnoobrazovkové zobrazenie dokumentu (canvas s bounding boxmi), vpravo je bočný panel s ovládacími prvkami.

\subsubsection{Krok 1 – Upload}

Komponent \texttt{UploadStep} umožňuje drag\,\&\,drop nahranie alebo výber súboru cez dialóg. Po odoslaní súboru na backend (\texttt{POST /analyze-document}) sa zobrazí výsledok OCR validácie: dokument je prijatý (badge \textit{Accepted}) alebo odmietnutý (badge \textit{Rejected}). V prípade odmietnutia môže používateľ nahrať iný súbor.

% [PLACEHOLDER: screenshot UploadStep s prijatým dokumentom — pipeline_step1.png]

\subsubsection{Krok 2 – Preprocessing}

Komponent \texttt{PreprocessingStep} zobrazí náhľad dokumentu a bočný panel s nastaveniami predspracovania. Posuvníky rotácie (−45° až +45°), jasu a kontrastu sa odrážajú okamžite v náhľade pomocou CSS transformácií. Prepínače \textit{Auto Deskew}, \textit{Binarize}, \textit{Denoise}, \textit{Shade Correction} a \textit{Remove Speckles} spúšťajú serverové predspracovanie (\texttt{POST /preprocess-image}); výsledok sa zobrazí v náhľade. Tlačidlo \textit{Reset all} obnoví všetky nastavenia na predvolené hodnoty.

% [PLACEHOLDER: screenshot PreprocessingStep — pipeline_step2.png]

\subsubsection{Krok 3 – Detection}

Komponent \texttt{DetectionStep} je najkomplexnejší krok. Po kliknutí na \textit{Run Detection} odošle predspracovaný obraz na backend (\texttt{POST /detect-boxes}), ktorý spustí YOLO inferenciu. Výsledné bounding boxy sa zobrazia na canvase farebne odlíšené podľa triedy.

Bočný panel zobrazuje accordion zoznam tried (section / pair / element) s ich bounding boxmi. Pre každú triedu je možné vybrať vlastný model (používateľ si môže nahrať vlastné váhy cez \texttt{POST /me/yolo-models}); predvolene sa použije serverový model. Používateľ môže:
\begin{itemize}
    \item kliknutím na bounding box na canvase ho vybrať a zvýrazniť v paneli,
    \item táhnutím posúvať alebo meniť veľkosť bounding boxov,
    \item vymazávať boxy (tlačidlo \faTrash),
    \item skrývať/zobrazovať triedy (ikona oka),
    \item premenovávať triedy, pridávať nové vlastné triedy,
    \item použiť \textit{Undo}/\textit{Redo} (história akcií).
\end{itemize}
Výsledky detekcie sú uložené do pipeline kontextu a prenesené do ďalšieho kroku.

% [PLACEHOLDER: screenshot DetectionStep s bounding boxmi — pipeline_step3.png]

\subsubsection{Krok 4 – OCR Review}

Komponent \texttt{ReviewStep} zobrazuje detegované páry a ich elementy v hierarchickom zozname. Po kliknutí na \textit{Run OCR} odošle backend požiadavku na Tesseract OCR (\texttt{POST /ocr-elements}) pre každý element; výsledky sa zobrazia ako transkripcia pri každom bounding boxe.

Používateľ môže:
\begin{itemize}
    \item reklasifikovať element na \textit{plain text}, \textit{cipher text} alebo \textit{symbol},
    \item manuálne opraviť OCR transkripciu,
    \item mazať alebo skrývať boxy.
\end{itemize}
Elementy sú automaticky zaradené do párov podľa geometrického prekryvu; párový zoznam je zoradený zvislo podľa polohy na dokumente.

% [PLACEHOLDER: screenshot ReviewStep s OCR transkripciou — pipeline_step4.png]

\subsubsection{Krok 5 – Symbol Match}

Komponent \texttt{ExportStep} (krok 5) sa zameriava na \textit{cipher text} elementy. Po kliknutí na \textit{Match Symbols} odošle backend požiadavku na symbolové priradenie (\texttt{POST /match-symbols}). Každý element zobrazí:
\begin{itemize}
    \item farebný indikátor stavu zhody (\texttt{matched} – zelená, \texttt{possible} – žltá, \texttt{not\_found} – červená, \texttt{new\_symbol} – fialová),
    \item najlepší kandidát z databázy s percentuálnou spoľahlivosťou,
    \item rozbaľovací zoznam top-3 kandidátov pre manuálny výber.
\end{itemize}
Používateľ môže priradenie potvrdiť, opraviť alebo pridať manuálnu poznámku.

% [PLACEHOLDER: screenshot ExportStep so symbolovým mapovaním — pipeline_step5.png]

\subsubsection{Krok 6 – Export}

Komponent \texttt{FinalExportStep} zostaví hierarchickú JSON štruktúru z bounding boxov (sekcie → páry → elementy) a zobrazí náhľad exportu. Tlačidlo \textit{Save to account} odošle výsledok na backend (\texttt{POST /export-pipeline}), kde sa uloží ako export dokumentu. Tlačidlo \textit{Download JSON} umožní priame stiahnutie súboru.

Exportný JSON obsahuje verziu formátu, zoznam sekcií, párov mimo sekcií, nepriradených elementov a súhrnné štatistiky.

% [PLACEHOLDER: screenshot FinalExportStep s JSON náhľadom — pipeline_step6.png]

\subsubsection{BBoxCanvas – interaktívne kreslenie bounding boxov}

Komponent \texttt{BBoxCanvas} je spoločný pre kroky 3–6 a zodpovedá za vykreslenie bounding boxov na SVG vrstve nad obrázkom dokumentu. Implementuje:
\begin{itemize}
    \item kreslenie nových boxov ťahaním myšou (kliknutie mimo existujúci box),
    \item presúvanie boxov (ťahanie za vnútro),
    \item zmenu veľkosti boxov (8 rukovätí na rohoch a hranách),
    \item zvýraznenie hover/selected boxu,
    \item škálovanie súradníc medzi zobrazenými pixelmi a reálnymi rozmermi obrázka.
\end{itemize}

\subsubsection{Správa stavu bounding boxov}

Celý stav bounding boxov (poloha, veľkosť, trieda, viditeľnosť, OCR transkripcia, stav symbolového mapovania) je riadený v hooku \texttt{useBoxEditor}. Hook zabezpečuje históriu akcií pre undo/redo, detekciu kolízií pri presúvaní a správu škálovacích faktorov pri zmene veľkosti okna prehliadača.

\subsection{Konfigurácia projektu a spúšťanie}

Vývojové prostredie je kontajnerizované pomocou Docker. Dockerfile.dev spustí React development server s hot-reload (\texttt{CHOKIDAR\_USEPOLLING=true}) na porte 3000. Konfiguračný súbor \texttt{package.json} definuje závislosti a skripty.

\subsection{Komunikácia s backendom}

Frontend komunikuje s backendovou službou prostredníctvom REST API pomocou natívneho \texttt{fetch} API. Základná URL backendu je definovaná v konštante \texttt{API\_BASE} (\texttt{REACT\_APP\_API\_URL} z prostredia). Všetky volania chránených endpointov obsahujú JWT token v hlavičke \texttt{Authorization}. Frontend je striktne oddelený od AI modulu a komunikuje s ním výhradne nepriamo cez backendovú vrstvu.
